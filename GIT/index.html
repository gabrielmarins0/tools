GIT

Instalando

Baixe o git em https://git-scm.com/download/win. No windows, use o comando winget install -id=Git.Git -e para instalar mais facilmente.

Instale o git usando as configurações padrões.

Verifique se o git foi instalado corretamente utilizando git -v no terminal.


Configurando usuário

git config --global user.name "Seu Nome" Para configurar o nome do usuário que ficará salvo nos commits.

git config --global user.email "Seu Email" Para configurar o email do usuário que ficará salvo nos commits.

--global deve ser usado caso seja deseje configurar para todos os projetos, remova para configurar apenas no diretorio atual.

Lembrando que este nome e email não tem haver com a autenticação do github, serve apenas para ficar registrado nos commits.


Iniciando um novo repositorio local

git init Para iniciar um novo repositorio

git branch -m "Nome da Branch" Para renomear a branch, normalmente é "master" e alterado para "main"


Configurando repositorio remoto

git remote add origin https://github.com/OWNER/REPOSITORY.git Para adicionar o repositorio remoto, o link do repositorio é obitido criando um repositorio no github

git remote set-url origin https://github.com/OWNER/REPOSITORY.git Para alterar o repositorio remoto

git push -u origin "Nome da Branch" Para enviar o repositorio para o servidor

Será solicitado realizar a autenticação do usuário no github para acesso ao repositorio remoto


Git Ignore

Crie um arquivo chamado .gitignore na pasta raiz do seu projeto

Adicione dentro deste arquivo o nome dos arquivos e pastas que deseja ignorar, por exemplo: node_modules, .ini, .env.

Os nomes de arquivos e pastas adicionados ao .gitignore não serao adicionados aos commits


Primeiro Commit

git add . Para adicionar todos os arquivos da pasta (exceto o .gitignore) ao proximo stage, caso deseje comitar apenas um arquivo,coloque o nome do arquivo em vez do ponto.

git status Para ver o status do estado atual do git

git commit -m "Descrição do commit" Para fazer o primeiro commit

git status Novamente para validar o commit


Verificando commits

git log Para ver todos os commits com detalhes

git log --oneline Para ver todos os commits sem detalhes


Desfazendo Commits

Verifique o número do commit com git log --oneline

git checkout "id do commit" Volta para o commit especificado, é possível ver como estava o projeto neste ponto

Use git checkout "Nome da Branch" para voltar para a branch mais recente. Exemplo: git checkout main

git revert "id do commit" Ira remover as alterações feitas no commit especificado até a branch mais recente, criando um novo commit.

git reset "id do commit" --hard Remove todos os commits após o commit especificado.


Criando Branches

git branch Para ver todas as branches, a brach com asterisco indica a branch atual

git branch "Nome da Branch" Cria uma nova branch com base na brach atual

git checkout "Nome da Branch" Vai para a branch especificada

git checkout -b "Nome da Branch" Cria uma nova branch e vai para ela

git branch -d "Nome da Branch" Remove a branch especificada


Fundindo Branches

git merge "Nome da Branch" Faz o merge da branch especificada com a branch atual
